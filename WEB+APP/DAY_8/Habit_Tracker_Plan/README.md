# Habit Tracker — Design Challenges (Day 8)

The **Day 8 Design Challenges** deliverable. A complete, from-scratch app plan for a **Daily
Habit Tracker**, produced by applying the full planning process from Section 28. It demonstrates
taking an idea from concept to a buildable blueprint.

## What is this app?

The **Habit Tracker** helps users build good habits and stay consistent. Users create daily
habits, check them off each day, and watch their streaks grow — turning small daily wins into
lasting motivation.

## The 5 design challenges covered

1. **Problem Statement** — who has the problem, the consequence, and how the app solves it.
2. **Feature List (MVP)** — a prioritized table: must-have (MVP) vs nice-to-have.
3. **User Flow** — how the user completes the main task (checking off a habit).
4. **Wireframes** — ASCII layouts for 3 screens (Home, Add Habit, Habit Details).
5. **Navigation Structure** — how all the screens connect (stack navigation).

See **[PLANNING.md](PLANNING.md)** for the complete details.

## MVP scope (v1)

Build the **must-have** features first:

- Add a habit
- View all habits (today)
- Mark a habit done
- Track streaks
- Delete a habit

> A reliable app that adds habits, checks them off, and shows streaks is genuinely useful on its
> own. **Start small and iterate.**
