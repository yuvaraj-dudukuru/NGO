# Personal Expense Tracker — Planning Lab (Day 8)

The **Day 8 Mobile Development Workflow Lab** deliverable. Since Day 8 is conceptual, this lab is
a **planning exercise**: plan an app from scratch by applying the full process from Section 28.
The result is a real, buildable app blueprint — a great portfolio artifact and the foundation for
coding in Flutter from Day 9.

## What is this app?

The **Personal Expense Tracker** helps users record their spending and see where their money
goes. Users quickly log each expense (amount, category, date), then view clear totals and a
breakdown — so they always know where their money is going.

## The planning process (the lab's 5 artifacts)

1. **Problem Statement** — who has the problem, the consequence, and how the app solves it.
2. **Requirements / Features** — a prioritized table (must-have = MVP, nice-to-have = later).
3. **User Flow** — how the user accomplishes the primary goal, screen by screen.
4. **Navigation Structure** — how all the screens connect.
5. **Wireframes** — ASCII layout sketches of the main screens.

See **[PLANNING.md](PLANNING.md)** for the complete details.

## MVP scope (v1)

Build the **must-have** features first:

- Add an expense (amount, category, date)
- View all expenses
- See total spending
- Delete an expense
- Categorize expenses

> A working app that records and totals expenses reliably beats an unfinished one crammed with
> charts and budgets. **Start small and iterate.**
