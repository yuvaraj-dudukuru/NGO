# App Analysis Demo (Day 8)

A **demo analysis** for the Day 8 *App Analysis Tasks*. It applies the day's mobile concepts —
app categories, device features, architecture layers, UI/UX, navigation patterns, and
cross-platform frameworks — to real, popular apps. Use it as a worked example / template for
your own analysis (ideally based on apps **you** personally use).

## What's in this folder?

| File          | Description                                            |
| ------------- | ------------------------------------------------------ |
| `ANALYSIS.md` | The full demo analysis covering all 5 tasks            |
| `README.md`   | This overview                                          |

## The 5 tasks covered

1. **Analyze a popular app** — WhatsApp: category, device features, architecture layers, UX strengths.
2. **Navigation patterns** — how WhatsApp's tabs, stack navigation, and FAB connect its screens.
3. **Compare two apps in a category** — WhatsApp vs Telegram (UI & UX differences).
4. **A poorly designed app** — three specific UX problems and concrete fixes.
5. **Flutter vs React Native** — which well-known apps use each framework.

See **[ANALYSIS.md](ANALYSIS.md)** for the complete write-up.

## How to use this

- Read it as a **model** for the depth and structure expected.
- Then redo each task with **apps you actually use** — your own observations make the analysis
  genuine and more valuable.
- Verify any framework/company claims against official sources, since these change over time.
