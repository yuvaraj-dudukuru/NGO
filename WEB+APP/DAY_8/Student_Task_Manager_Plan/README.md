# Student Task Manager — Planning Project (Day 8)

A **planning project**, not a coding project. The goal of Day 8 is to learn how to *plan* an
app properly before building it. The output here is the blueprint for the Flutter app that
will be built starting Day 9.

## What is this app?

The **Student Task Manager** is a productivity app that helps students organize their
assignments and tasks — add tasks, set priorities and due dates, mark tasks complete, and see
everything they need to do at a glance. It's a great first app: useful, clearly scoped, and
built from familiar components.

## The planning process (6 steps)

1. **Problem Statement** — define the problem, who has it, and why it matters.
2. **Features** — list capabilities, prioritized into *must-have* vs *nice-to-have*.
3. **Screens** — identify the screens the app needs.
4. **User Flow** — map how users move through screens to accomplish goals.
5. **Wireframes** — sketch each screen's layout (boxes and labels, no styling).
6. **Navigation Structure** — define how screens connect.

See **[PLANNING.md](PLANNING.md)** for the complete details.

## Scope for the first version (v1)

Build the **must-have** features first:

- Add a task
- View all tasks
- Mark a task complete
- Delete a task
- Set a due date

> A working app that does a few things well beats an unfinished app that tries to do
> everything. **Start small and iterate.**
