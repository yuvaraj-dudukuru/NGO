# Data Dictionary — `sales_clean.csv`

The cleaned, analysis-ready dataset for the GreenCart capstone. One row = one order.
All data is **synthetic** (generated by [`generate_data.js`](generate_data.js)); no real or personal data.

| Column | Type | Description | Example |
|---|---|---|---|
| `order_id` | string | Unique order identifier | `ORD-1042` |
| `order_date` | date (ISO) | Date the order was placed | `2025-03-14` |
| `month` | integer | Month number 1–12 (derived from `order_date`) | `3` |
| `region` | category | Sales region: North / South / East / West | `South` |
| `category` | category | Product category: Electronics / Apparel / Home / Grocery / Sports | `Electronics` |
| `segment` | category | Customer segment: Premium / Regular / Basic | `Premium` |
| `customer_id` | string | Pseudonymous customer key (no personal data) | `C0231` |
| `units` | integer | Units purchased in the order | `3` |
| `unit_price` | float ($) | Price per unit before discount | `249.90` |
| `discount_pct` | float (%) | Discount applied to the order, 0–55% | `12.5` |
| `revenue` | float ($) | Net revenue = `unit_price × units × (1 − discount_pct/100)` | `655.99` |
| `cost` | float ($) | Cost of goods for the order | `554.78` |
| `profit` | float ($) | `revenue − cost` | `101.21` |
| `margin_pct` | float (%) | `profit ÷ revenue × 100` | `15.4` |
| `is_new_customer` | 0/1 | 1 if this is the customer's first order in the dataset, else 0 | `1` |

## Notes & conventions

- **Currency**: all monetary values are in US dollars.
- **Grain**: order level. Aggregate by `month`, `region`, `category`, or `segment` for analysis.
- **Derived fields**: `month`, `revenue`, `cost`, `profit`, `margin_pct`, and `is_new_customer`
  are computed during cleaning (see [`../analysis/methodology.md`](../analysis/methodology.md)).
- **Validation guarantees**: discounts within 0–55%; `units`, `revenue`, `cost` non-negative;
  no duplicate `order_id`; every `customer_id` is consistent in region and segment.
