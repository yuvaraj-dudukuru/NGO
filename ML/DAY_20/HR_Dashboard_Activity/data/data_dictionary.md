# Data Dictionary — `employees.csv`

The cleaned employee dataset for the Northwind HR hands-on activity. One row = one employee.
All data is **synthetic** (generated by [`generate_data.js`](generate_data.js)); no real or personal data.
Employee IDs are pseudonymous keys.

| Column | Type | Description | Example |
|---|---|---|---|
| `employee_id` | string | Pseudonymous employee key (no personal data) | `E1042` |
| `department` | category | Engineering / Sales / Operations / Customer Support / Marketing / Finance / HR | `Engineering` |
| `location` | category | HQ / Remote / Regional-North / Regional-South | `HQ` |
| `gender` | category | Female / Male / Non-binary · Other (synthetic) | `Female` |
| `role_level` | category | Junior / Mid / Senior / Lead | `Mid` |
| `hire_date` | date (ISO) | Date the employee was hired | `2022-06-14` |
| `exit_date` | date (ISO) or blank | Date the employee left; blank if still active | `2025-10-03` |
| `status` | category | `Active` or `Left` | `Active` |
| `exit_reason` | category or blank | Reason given on exit (blank if active) | `Career growth` |
| `tenure_months` | integer | Tenure in months (to exit date, or to 2025-12-31 if active) | `42` |
| `tenure_band` | category | `0-18 mo` / `18-36 mo` / `3-5 yrs` / `5+ yrs` | `3-5 yrs` |
| `engagement_score` | float (1–5) | Latest engagement survey score | `3.1` |
| `performance_rating` | float (1–5) | Latest performance rating | `3.8` |
| `salary_k` | integer ($K) | Annual base salary in thousands | `88` |

## Conventions & how metrics are computed

- **Analysis date**: 2025-12-31. "Active" = `exit_date` is blank.
- **Attrition rate** = exits in period ÷ (active + exits in period).
- **Monthly headcount** = employees hired on/before month-end and not yet exited by month-end.
- **Avg tenure / engagement** are computed over **active** employees.
- **Validation guarantees**: every employee has a valid `hire_date ≤ exit_date`; engagement and
  performance bounded to 1–5; `status` consistent with presence/absence of `exit_date`.

See [`../analysis`](../) note in the README for the full methodology and reproducibility.
